# Delivery Scheduling with Pandas and OR-Tools

## Project Goal

This project prepares food delivery order data for a future scheduling optimization model.

## Current Work

- Read CSV data
- Inspect dataset
- Check missing values
- Filter cancelled orders
- Filter by distance
- Filter by order value
- Add delivery cost column
- Analyze orders by zone using groupby
- Save cleaned dataset
## Driver Data Cleaning

The driver dataset was cleaned by:

- checking missing values
- removing incomplete records
- filtering unavailable drivers
- filtering drivers with zero capacity
- organizing drivers by delivery zone

# Results

After cleaning and analyzing the food delivery datasets, several insights were obtained.

## Order Analysis

* Cancelled orders were removed from the dataset.
* Orders outside the allowed delivery distance were filtered out.
* High-priority and valid delivery requests were identified.
* Delivery costs were estimated based on travel distance.

## Driver Analysis

* Unavailable drivers were removed from the dataset.
* Drivers with zero delivery capacity were excluded.
* A clean set of active drivers was prepared for scheduling.

## Demand Analysis

* The number of orders was calculated for each delivery zone.
* High-demand zones were identified.
* Revenue generated by each zone was analyzed.
* Average delivery distance was computed for every zone.

## Capacity Analysis

* Driver capacity was calculated for each zone.
* Demand was compared against available driver capacity.
* Understaffed and overstaffed zones were identified.
* Workload distribution across zones was evaluated.

## Driver-Order Matching

* Orders were matched with drivers operating in the same zone.
* Candidate assignments were generated for future optimization.
* Priority scores were calculated to identify attractive delivery assignments.

## Scheduling Preparation

The cleaned and processed datasets are now ready for optimization using Google OR-Tools. The data preparation stage successfully transformed raw business data into a structured format suitable for scheduling and decision-making.

## Conclusion

Pandas was used to perform data cleaning, filtering, aggregation, matching, and analysis. These preprocessing steps created a reliable foundation for building a delivery scheduling optimization model. The resulting datasets can now be used to assign orders to drivers while respecting operational constraints and maximizing efficiency.
